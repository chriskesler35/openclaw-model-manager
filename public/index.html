Remote Connections

Add Remote Connection

📋 Before connecting to a remote machine:
  1. Install Model Manager on the remote machine — git clone the repo, run npm install, then node server.js
  2. Open firewall ports on the remote machine for incoming connections:
    • 18800 (Model Manager) — required for remote management
    • 11434 (Ollama) — needed to see local AI models
    • 18789 (OpenClaw Gateway) — needed for gateway health checks
    Example firewall commands
    Windows (admin PowerShell) — allow from Tailscale only: 
    netsh advfirewall firewall add rule name="Model Manager" dir=in action=allow protocol=TCP localport=18800 remoteip=100.64.0.0/10
netsh advfirewall firewall add rule name="Ollama" dir=in action=allow protocol=TCP localport=11434 remoteip=100.64.0.0/10
netsh advfirewall firewall add rule name="OpenClaw Gateway" dir=in action=allow protocol=TCP localport=18789 remoteip=100.64.0.0/10
    Linux (ufw): 
    sudo ufw allow from 100.64.0.0/10 to any port 18800,11434,18789 proto tcp
  3. Use the remote machine's IP — Tailscale IP (100.x.x.x), LAN IP (192.168.x.x), or hostname
